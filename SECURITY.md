## Reporting a Vulnerability

If you discover a security vulnerability, an unauthorized system override, or an exploit within any of the 7 pillars (including transaction loops or data verification structures), **do not open a public GitHub issue.** Public issues disclose system vulnerabilities to bad actors.

Please report your findings directly via our automated secure routing channels:

1. **Secure Email Portal:** Submit a detailed exploit brief to `security@nawi-empire.com` (or your dedicated system infrastructure address).
2. **Encrypted Reporting:** Include an explanation of the steps required to replicate the vulnerability, along with sample payload footprints if applicable.

### What to Expect
* **Acknowledgment:** The automated network security system will acknowledge receipt of your report within 24 hours.
* **Triage & Patching:** Legitimate system vulnerabilities will be triaged and patched out-of-band by our automated deployment infrastructure through `border-control.js` without system downtime.
* **Confidentiality:** All validated security disclosures are handled strictly within the automated protocol layer to maintain absolute system integrity.
